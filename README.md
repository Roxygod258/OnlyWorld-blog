# OnlyWorld 个人博客

一个无需安装依赖、可以直接在本地打开的个人博客。内容包括学习笔记、杂谈、个人主页、音乐播放器和本地外观设置。

## 打开网站

直接双击 `index.html`。修改文件后，在浏览器中按 `Ctrl + F5` 强制刷新。

顶部日期显示格式为 `年/月/日  时间  农历日期`，例如 `2026/07/17  22:40:00  农历六月初四`。

## 两种修改方式的区别

### 页面内设置

适合目前只在自己的电脑上浏览：

- 设置页面：点击导航栏的齿轮图标进入独立设置页，可以调节遮罩、背景柔化、内容不透明度、导航不透明度、樱花飘落和音乐播放音量。
- 单曲播放：进入“音乐”页面后点击播放，歌曲会持续单曲循环。

这些设置保存在当前浏览器中，不会自动写回项目文件。清除浏览器网站数据后，本地修改会消失；把网站发布到网上后，其他访客也看不到这些本地设置。

### 修改内容文件

适合以后公开发布。文件中的内容会成为所有访客看到的默认内容：

- `site-content.js`：欢迎页、个人资料、壁纸、外观默认值和音乐配置。
- `site-posts.js`：学习笔记和杂谈文章。
- `styles.css`：颜色、字号和布局。

建议使用记事本或代码编辑器打开文件，并以 UTF-8 编码保存。

## 修改个人资料

网站不提供个人资料编辑按钮。打开 `site-content.js`，修改 `profile` 内引号中的内容：

- `avatar`：头像路径，例如 `"assets/avatar.jpg"`。
- `greeting`：个人主页标题。
- `bio`：个人简介。
- `email`：邮箱地址。
- `birthday`：生日，例如 `"7.21"`，显示在邮箱正下方。
- `github`：GitHub 用户名，不需要填写完整网址。
- `direction`：当前学习方向。
- `topics`：主要记录主题。
- `status`：当前状态。
- `website`、`belief`：个人主页下方的介绍文字。

公开头像的设置方法：

1. 把头像图片放入 `assets` 文件夹，例如 `assets/avatar.jpg`。
2. 将 `site-content.js` 中的 `avatar: ""` 改成 `avatar: "assets/avatar.jpg"`。
3. 保存文件并刷新网页。

个人资料只读取 `site-content.js`，浏览器中以前保存过的个人资料不会再覆盖文件内容。

## 修改欢迎页

网站不再显示欢迎页编辑按钮。要修改欢迎页文字，请编辑 `site-content.js` 中的 `welcome`：

- `kicker`：标题上方的小字。
- `title`：主标题。
- `subtitle`：欢迎语。
- `button`：进入按钮文字。

如果以前通过页面保存过欢迎文字，旧数据可能仍会覆盖文件内容。打开浏览器开发者工具，在控制台执行下面一行，然后刷新：

```js
localStorage.removeItem("onlyworld-welcome-copy");
```

## 添加学习笔记

打开 `site-posts.js`，找到 `notes: [`。复制其中一整段文章对象，粘贴到同一数组中，再修改内容：

```js
{
  id: "unique-note-id",
  title: "文章标题",
  date: "2026-07-17",
  tags: ["标签一", "标签二"],
  summary: "显示在文章列表中的摘要。",
  readTime: "5 分钟",
  content: `
    <p>第一段正文。</p>
    <h2>小标题</h2>
    <p>第二段正文。</p>
  `,
},
```

注意：

- `id` 必须是唯一的英文标识，不能和其他文章重复。
- 日期必须使用 `YYYY-MM-DD` 格式。
- 正文放在一对反引号之间。
- 每篇文章结束的 `},` 不要删除。
- 段落使用 `<p>文字</p>`，小标题使用 `<h2>标题</h2>`。
- 代码块可以使用 `<pre><code>代码</code></pre>`。

网站会自动更新首页目录、搜索和日期归档，不需要修改其他文件。

## 添加杂谈

打开 `site-posts.js`，找到 `thoughts: [`，复制一篇现有杂谈并修改。杂谈必须保留：

```js
kind: "thought",
```

其他字段与学习笔记相同。保存后，文章会自动显示在杂谈区、侧栏目录、搜索和日期归档中。

## 自定义壁纸

网站页面不再提供壁纸上传按钮。壁纸由你在本地项目中设置，发布后访客只能看到设置好的默认壁纸。

1. 把图片放入 `assets` 文件夹，例如 `assets/wallpaper.jpg`。
2. 打开 `site-content.js`，找到或添加 `appearance`：

```js
appearance: {
  wallpaper: "assets/wallpaper.jpg",
  shade: 32,
  blur: 0,
  contentOpacity: 70,
  navOpacity: 88,
  musicVolume: 80,
  sakura: true,
},
```

3. 保存文件并按 `Ctrl + F5` 刷新网页。

各字段含义：

- `wallpaper`：壁纸的本地路径，也可以使用合法的 HTTPS 图片地址；留空时使用网站自带背景。
- `shade`：浅色遮罩强度，范围建议为 `10` 到 `72`。
- `blur`：背景柔化像素，范围建议为 `0` 到 `16`。
- `contentOpacity`：正文和目录不透明度，范围为 `38` 到 `100`。
- `navOpacity`：顶部导航不透明度，范围为 `45` 到 `100`。
- `musicVolume`：默认音乐音量，范围为 `0` 到 `100`。
- `sakura`：是否默认开启樱花飘落，填写 `true` 或 `false`。

导航栏齿轮设置中的选项只保存在当前浏览器，不会改写 `site-content.js`。点击“恢复默认”只恢复透明度、柔化、音量和樱花开关，不会清除或更换当前壁纸。

## 添加照片

把照片放入 `assets` 文件夹，然后在 `site-content.js` 的 `photos` 数组中添加：

```js
photos: [
  {
    src: "assets/photo-01.jpg",
    alt: "照片内容描述",
    caption: "照片标题",
    date: "2026-07-17",
  },
],
```

## 设置音乐播放器

音乐页面使用本地单曲循环播放器，支持专辑封面、播放/暂停、进度和音量控制。进入网站和音乐页面时默认不会播放，必须由访客主动点击播放。

歌曲播放结束后会自动从头开始。开始播放后，即使切换到首页、文章、设置或其他导航页面，音乐也会继续播放；再次进入音乐页可以继续控制当前进度。

### 设置循环歌曲

1. 将你有权使用的音频文件放入 `assets`，例如 `assets/my-song.mp3`。
2. 将歌曲封面放入 `assets`，例如 `assets/my-song.jpg`。
3. 打开 `site-content.js`，填写：

```js
music: {
  title: "歌曲名称",
  artist: "歌手名称",
  src: "assets/my-song.mp3",
  cover: "assets/my-song.jpg",
},
```

保存后按 `Ctrl + F5` 刷新。建议使用 MP3 格式以获得更好的浏览器兼容性；`cover` 可以留空，播放器会显示音乐符号占位。

## 文件说明

- `index.html`：网页结构和各个弹窗。
- `styles.css`：全站视觉样式与响应式布局。
- `site-content.js`：欢迎页、个人资料、壁纸、外观和音乐默认内容。
- `site-posts.js`：学习笔记和杂谈文章。
- `script.js`：页面交互程序，日常更新内容通常不需要修改。

## 发布网站

这是纯静态网站，可以部署到 GitHub Pages、Cloudflare Pages、Netlify 等静态托管平台。发布前建议复制整个文件夹作为备份，并在电脑和手机尺寸下各检查一次。
